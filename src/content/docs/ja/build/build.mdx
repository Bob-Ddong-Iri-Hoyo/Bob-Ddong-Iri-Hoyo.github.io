このドキュメントは、Bob-Ddong-Iri-Hoyo ランチャーで使用する Wine と DXMT の構成要素を自分でビルドしたい、またはビルド環境を理解したいユーザー向けに基準情報を整理したものです。
ランチャーだけを使用する場合、このドキュメントに従ってビルドする必要はありません。

## クロスコンパイラ構成

ビルド過程では、目的の異なる 2 種類のツールチェーンが使われます。

- Wine のビルドには、Windows ターゲットのバイナリを作成するための `mingw` ツールチェーンが必要です。
- DXMT のビルドには、Metal 変換レイヤーをビルドするための `llvm` ベースのツールが必要です。

Wine 用の `mingw` ツールチェーンは、別途用意された toolchain をダウンロードして使用します。
DXMT 用の `llvm` は、DXMT のビルド手順に合わせるため、別途ビルドする方式を基準にしています。
両方を 1 つの共通ツールチェーンに統合することもできますが、現在のドキュメントでは DXMT チュートリアルの流れに沿って、Wine と DXMT のビルド手順を分けて説明します。

## ビルド文書の読み方

このページは、すべてのビルドコマンドを並べるのではなく、どの構成要素がなぜ必要なのかを理解することに焦点を置いています。
実際のビルドスクリプトや詳細なコマンドは、関連リポジトリの README とビルド文書もあわせて確認してください。

ビルドが失敗した場合は、まず使用している Wine バージョン、DXMT バージョン、`mingw` と `llvm` のバージョンがこのドキュメントの基準と一致しているか確認することをおすすめします。
互換レイヤーのプロジェクトは依存関係のバージョンに大きく影響されるため、小さなバージョン差でもビルドエラーや実行時問題につながることがあります。
